# AIS ERP Suite 双核心与供应链制造 Agent 自动化开发计划

## 1. 文档目标

本文档用于指导 AIS ERP Suite 后续升级为“双核心一体化 ERP”：

- 财务管理中心：负责财务做账、凭证、总账、应收应付、出纳、成本、资产、薪资、报表和期末处理。
- 供应链制造中心：负责订单、采购、库存、生产、APS、委外、返工、追溯、线边仓、设备和统计报表。
- Agent 自动化体系：让 Agent 在权限、证据、dry-run、审批和审计约束下代替人工完成重复性供应链制造工作。

目标不是把财务和供应链拆成两个数据孤岛，而是形成“业务单据 -> 库存台账 -> 成本核算 -> 财务凭证 -> 报表分析”的业财一体化闭环。

## 2. 总体产品架构

### 2.1 两个业务中心

系统面向用户呈现为两个一级业务中心：

| 中心 | 定位 | 主要用户 |
| --- | --- | --- |
| 财务管理中心 | 做账、审核、记账、结账、出报表、对账和财务风控 | 会计、财务主管、出纳、审计人员 |
| 供应链制造中心 | 订单、采购、库存、生产、排程、委外、返工、追溯和成本前置数据 | 采购、仓库、计划、生产、PMC、车间主管 |

### 2.2 共享底座

两个中心共用同一套底层能力：

- 账套、组织、会计期间。
- 用户、角色、权限、数据范围。
- 客户、供应商、物料、仓库、部门、员工、资产。
- 附件、OCR 结果、证据引用、审计日志。
- 审批流、状态机、操作回放。
- Agent Action、Agent Draft、dry-run、人工确认。
- 凭证生成规则、业务来源链路、报表取数规则。

### 2.3 数据互通原则

- 业务单据负责记录业务事实：订单、入库、出库、领料、完工、返工、委外、盘点。
- 库存台账负责记录物料数量、仓库、库位、批次、序列号和成本层。
- 成本核算负责归集材料、人工、折旧和制造费用。
- 财务凭证负责记录金额、科目、借贷方向、辅助核算和会计期间。
- 报表负责从凭证、余额、台账和业务链路中生成经营结果。
- Agent 只能通过稳定 Tool API 操作业务，不直接绕过业务规则写底层数据。

## 3. 财务管理中心设计

### 3.1 一级菜单

- 财务工作台
- 基础设置
- 凭证管理
- 总账管理
- 应收管理
- 应付管理
- 出纳与银行
- 成本与存货核算
- 固定资产
- 薪资核算
- 财务报表
- 期末处理
- 财务审计

### 3.2 核心能力

#### 财务工作台

- 展示待审核凭证、待记账凭证、异常凭证、应收应付预警和结账检查项。
- 展示供应链制造中心推送来的待生成凭证、待对账、待结转任务。
- 支持按账套、期间、风险等级、业务来源筛选。

#### 凭证管理

- 支持凭证录入、凭证列表、凭证审核、凭证记账、冲销、反审核申请。
- 支持从采购、销售、库存、生产、成本、薪资、固定资产自动生成凭证草稿。
- 所有凭证必须能反查来源单据、Agent Action、审批记录和证据。

#### 总账管理

- 支持科目余额表、明细账、总账、辅助核算账。
- 支持从账表下钻到凭证，再反查业务单据。
- 支持库存台账与总账对账。

#### 应收应付

- 采购入库、采购发票、付款形成应付链路。
- 销售发货、销售发票、收款形成应收链路。
- 支持核销、账龄分析、付款计划、回款计划。
- 支持 Agent 生成核销建议和逾期提醒。

#### 成本与存货核算

- 支持库存余额、成本层、移动平均、FIFO、个别计价。
- 支持生产领料成本、完工入库成本、成本分摊和成本凭证。
- 支持零数量残值、负库存、批次成本异常检查。

#### 财务报表

- 支持资产负债表、利润表、现金流量表、管理分析报表。
- 报表数据必须可追溯到科目余额、凭证和业务单据。
- Agent 可以生成报表解读草稿，但正式发布必须人工确认。

#### 期末处理

- 支持结账检查、损益结转、成本结转、期间关闭、期间锁定。
- 关闭期间后，供应链制造中心不得在该期间生成正式影响财务的数据。

## 4. 供应链制造中心设计

### 4.1 一级菜单

- 生产工作台
- 订单管理
- APS 管理
- 采购管理
- 库存管理
- 生产管理
- 产品管理
- 设备管理
- 委外管理
- 追溯规则
- 线边仓管理
- 统计报表
- Agent 任务队列

### 4.2 核心页面

#### 生产工作台

- 展示今日订单、缺料预警、采购逾期、工单进度、库存异常、待领料、待完工、待审批。
- 展示供应链总控 Agent 生成的每日任务摘要。
- 支持从异常项直接跳转到订单、物料、仓库、工单或 Agent Action。

#### 订单管理

- 销售订单、采购订单、订单状态、订单履约进度。
- 支持订单到采购、库存、生产和财务的来源链路。
- 支持 Agent 检查订单是否可交付、是否缺料、是否需要采购。

#### APS 管理

- 主生产计划、计划详情、人工排程、计划状态跟踪。
- 第一版以规则型排程和人工调整为主，不做复杂自动排程算法。
- 支持 Agent 根据订单、库存、BOM、产能生成排程草稿。

#### 采购管理

- 采购申请、采购订单、采购入库、采购退货、采购发票。
- 支持根据缺料清单生成采购建议。
- 支持供应商推荐、价格异常检查、交期风险提醒。

#### 库存管理

- 物料库存、生成发货单、出入库单、调拨单、盘点、库存台账。
- 仓库分类包括主仓、退货仓、委外仓、报废仓、部门仓、线边仓。
- 库存查询必须支持物料、规格、属性、类别、BOM、仓库、库位、批次、可用量、锁定量。

#### 生产管理

- 工单管理、工单详情、领料申请、完工入库、返工计划、新增返工、返工审批、关单审批。
- 支持从生产计划生成工单。
- 支持从工单和 BOM 生成领料申请。
- 支持完工入库后触发成本核算。

#### 产品管理

- 物料档案、BOM、产品工艺、版本管理。
- BOM 必须能被 APS、工单、领料、成本和追溯复用。

#### 委外管理

- 委外发料、委外收货、委外结算、委外在途查询。
- 支持产能不足或工序外协时生成委外建议。

#### 追溯规则

- 支持按批次、序列号、工单、销售订单、采购订单追溯。
- 支持正向追溯和反向追溯。
- 支持异常批次影响范围分析。

#### 线边仓管理

- 线边库存、补料申请、退料、消耗记录。
- 支持按产线、工单、班组、物料查询线边库存。

#### 统计报表

- 库存报表、采购报表、生产报表、订单履约报表、委外报表、追溯报表。
- 报表支持下钻到明细单据和 Agent Action。

## 5. 供应链制造 Agent 工作群

### 5.1 设计原则

- Agent 不是全局聊天框，而是一组虚拟业务岗位。
- Agent 从具体业务页面带上下文启动。
- Agent 通过 Tool API 调用业务能力，不依赖页面 DOM。
- 所有写操作先 dry-run，再保存草稿，再提交审批或人工确认。
- 高风险动作必须人工审批后执行。
- 每个 Agent 动作必须可回放、可解释、可审计、可追溯。

### 5.2 Agent 角色

#### 供应链总控 Agent

职责：

- 汇总订单、库存、采购、工单、缺料、成本和审批状态。
- 生成每日供应链待办。
- 识别缺料、超期、库存异常、工单逾期、采购未到货。
- 将任务派发给采购 Agent、库存 Agent、APS Agent、生产工单 Agent。

典型输出：

- 今日供应链风险清单。
- 今日待处理任务。
- 需人工审批的高风险事项。

#### APS 排产 Agent

职责：

- 根据销售订单、库存、BOM、产能生成主生产计划草稿。
- 检查排程冲突和缺料风险。
- 建议开工日期、完工日期、优先级。
- 生成建议工单草稿。

第一版规则：

- 订单交期优先。
- 库存可用量优先。
- 缺料订单标记为待采购。
- 超期订单标记为高风险。
- 人工可以调整排程结果。

#### 采购 Agent

职责：

- 根据缺料清单生成采购申请或采购订单草稿。
- 根据历史供应商、价格、交期推荐供应商。
- 检查采购订单价格异常、交期异常、供应商风险。
- 跟踪采购到货进度。
- 生成催货清单。
- 根据采购入库和采购发票生成应付确认建议。

#### 库存 Agent

职责：

- 查询物料库存、可用量、锁定量、批次、库位。
- 根据销售订单生成发货单草稿。
- 根据采购到货生成入库单草稿。
- 检查库存不足、负库存、批次过期、呆滞物料、成本异常。
- 生成盘点任务和盘点调整建议。

#### 生产工单 Agent

职责：

- 根据生产计划生成工单草稿。
- 根据工单和 BOM 自动生成领料申请。
- 检查领料是否超 BOM。
- 检查工单是否缺料。
- 根据完工报告生成完工入库草稿。
- 建议关单、返工或异常处理。

#### 返工 Agent

职责：

- 根据质检异常、客户退货或生产异常生成返工计划草稿。
- 自动关联原销售订单、原生产工单、原批次。
- 计算返工所需物料。
- 生成返工领料建议和返工审批材料。
- 返工完成后生成入库、报废或成本调整建议。

#### 委外 Agent

职责：

- 根据产能不足、工序外协规则或计划冲突生成委外建议。
- 生成委外发料单草稿。
- 跟踪委外在途物料。
- 生成委外收货单草稿。
- 检查委外损耗异常。
- 生成委外结算和应付建议。

#### 追溯 Agent

职责：

- 根据物料批次追溯采购来源、入库单、领料单、工单、成品批次、销售发货。
- 根据销售订单反查生产批次和原材料批次。
- 生成追溯报告。
- 异常批次出现时生成影响范围清单。

#### 成本核算 Agent

职责：

- 检查工单领料成本。
- 汇总人工、折旧、制造费用。
- 生成成本分摊 dry-run。
- 检查零数量有金额、负库存、成本层异常。
- 生成成本凭证草稿。
- 生成库存台账与总账对账报告。

## 6. Agent 自动化风险分级

| 风险等级 | 可自动执行范围 | 是否需要人工确认 | 示例 |
| --- | --- | --- | --- |
| 低风险 | 查询、分析、报告、异常检测 | 不需要 | 库存查询、缺料清单、逾期清单、追溯报告 |
| 中风险 | 生成业务草稿、生成建议、提交待办 | 需要人工确认后保存 | 采购订单草稿、领料单草稿、完工入库草稿、返工计划草稿 |
| 高风险 | 正式影响库存、成本、财务、付款、结账 | 必须审批后执行 | 正式出库、正式入库、成本结转、凭证记账、库存调整、付款、关单 |

系统必须保证：

- Agent 不能绕过权限。
- Agent 不能绕过期间关闭。
- Agent 不能绕过库存不足、批次规则、成本规则、审批规则。
- Agent dry-run 结果与正式执行结果差异必须可解释。

## 9. 典型自动化流程

### 9.1 缺料到采购自动化

1. Agent 读取销售订单、生产计划和工单需求。
2. 展开 BOM。
3. 查询库存可用量、锁定量、在途采购。
4. 计算缺料数量。
5. 生成缺料清单。
6. 根据供应商历史价格和交期生成采购建议。
7. 生成采购申请或采购订单草稿。
8. 提交采购员确认。

### 9.2 生产计划到工单自动化

1. Agent 读取销售订单和预测需求。
2. 查询库存和 BOM。
3. 生成主生产计划草稿。
4. 检查物料齐套情况。
5. 生成工单草稿。
6. 生成预计领料清单。
7. 提交生产主管确认。
8. 审批后下达工单。

### 9.3 工单到完工入库自动化

1. Agent 检查工单状态。
2. 检查是否已领料。
3. 读取完工数量和质检结果。
4. 生成完工入库草稿。
5. 更新库存 dry-run。
6. 触发成本核算 Agent。
7. 成本确认后生成成本凭证草稿。

### 9.4 库存异常巡检

1. 每天自动检查负库存。
2. 检查库存金额异常。
3. 检查批次过期。
4. 检查长期呆滞物料。
5. 检查仓库可用量不足。
6. 生成异常清单。
7. 推送到生产工作台和 Agent 任务队列。

### 9.5 追溯报告自动化

1. Agent 接收物料编码、批次号、销售订单号或工单号。
2. 查询采购入库、库存移动、领料、工单、完工入库、销售发货。
3. 生成正向或反向追溯链。
4. 输出受影响订单、批次、库存和客户清单。
5. 生成追溯报告草稿。

## 12. 开发阶段

### Phase 1: 双核心导航和信息架构

- 重组菜单为财务管理中心和供应链制造中心。
- 建立供应链制造中心顶部模块和左侧流程。
- 保留旧路由并做兼容跳转。
- 修复供应链相关菜单、页面标题、状态文案的中文乱码。

### Phase 2: 供应链制造工作台

- 新增生产工作台。
- 展示订单、采购、库存、工单、成本、审批和异常摘要。
- 新增供应链工作台聚合接口。
- 接入供应链总控 Agent 摘要。

### Phase 3: Agent 基础能力扩展

- 扩展 Agent Tool Registry。
- 新增供应链制造 Agent 工具。
- 所有工具支持 dry-run。
- 新增 Agent 任务队列页面。
- 打通 Agent Action replay 和供应链单据来源链路。

### Phase 4: 第一批高价值 Agent

优先实现：

1. 缺料分析 Agent。
2. 采购建议 Agent。
3. 库存异常 Agent。
4. 工单领料 Agent。
5. 完工入库与成本 Agent。

### Phase 5: 计划、返工、委外、追溯 Agent

- APS 排产 Agent。
- 返工 Agent。
- 委外 Agent。
- 追溯 Agent。
- 线边仓补料 Agent。

### Phase 6: 业财一体化闭环

- 供应链制造单据生成财务凭证草稿。
- 成本核算 Agent 生成成本凭证草稿。
- 财务中心审核、记账、结账。
- 存货台账与总账自动对账。
- 报表支持从财务结果反查到业务单据和 Agent Action。

## 13. 测试计划

### 13.1 导航测试

- 财务管理中心和供应链制造中心可访问。
- 供应链顶部模块和左侧流程显示正确。
- 旧路由仍可访问或正确跳转。
- 中文菜单、页面标题、状态文案显示正常。

### 13.2 页面测试

- 生产工作台可展示供应链摘要。
- 物料库存、生成发货单、主生产计划、工单管理、领料申请、返工、委外、线边仓、追溯规则均可访问。
- Agent 任务队列可筛选、查看、审批、执行、回放。

### 13.3 API 测试

- 新增聚合接口返回字段稳定。
- Agent 工具支持 dry-run。
- Agent 工具不能绕过权限、期间、库存和成本规则。
- 现有采购、销售、库存、生产、成本接口不回归。

### 13.4 业务场景测试

- 销售订单 -> 缺料分析 -> 采购建议 -> 采购入库 -> 应付建议。
- 销售订单 -> 库存可用量检查 -> 发货单草稿 -> 出库。
- 主生产计划 -> 工单 -> 领料 -> 完工入库 -> 成本试算 -> 成本凭证草稿。
- 盘点差异 -> 调整建议 -> 审批 -> 库存调整 -> 财务凭证草稿。
- 批次异常 -> 追溯报告 -> 影响范围清单。

### 13.5 风险控制测试

- 库存不足时，Agent 只能生成异常提示或采购建议，不得正式出库。
- 期间关闭时，Agent 不得生成正式影响财务的数据。
- 高风险动作未审批时，Agent 不得执行。
- Agent 草稿被人工修改后，必须记录修改差异。
- Agent 执行动作必须能回放。

## 14. 验收标准

- 财务管理中心和供应链制造中心结构清晰，数据互通。
- 供应链制造中心的关键页面均具备上下文 Agent 入口。
- Agent 能完成查询、分析、草稿生成、dry-run、提交确认、提交审批、审批后执行的完整闭环。
- Agent 不能绕过业务规则、权限、期间和审批。
- 每个 Agent 动作都有证据、来源、dry-run、审批、执行和回放记录。
- 财务凭证可以反查到业务单据、Agent Action 和证据。
- 供应链单据可以反查到财务凭证、库存台账和成本结果。
- 第一批高价值 Agent 能明显减少人工重复操作。
