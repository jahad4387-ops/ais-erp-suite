## 7. 会计期间测试

1. 打开“基础档案 -> 会计期间”。
2. 确认 2026 年第 1 期状态为 `open`。
3. 对第 2 期执行 Open。
4. 对第 2 期执行 Set Current。
5. 对第 2 期执行 Close 或 Lock。
6. 返回第 1 期继续做凭证闭环。

预期结果：

- 任意时间只能有一个当前期间。
- open 期间允许凭证操作。
- closed 或 locked 期间不能新增、修改、删除、提交、审核、记账凭证。
- 若期间存在未记账凭证，Close 应被阻止。

1. 打开“Agent 中心”。
2. 输入业务文本：`购买办公用品 500 元，银行支付`。

1. 检查 AI 建议的科目和金额。
2. 点击人工确认或 Convert。
3. 打开“总账 -> 凭证管理”查看新凭证。

预期结果：

- 只有人工确认后才生成正式凭证草稿。
- 正式草稿仍需要提交、审核、记账。
- 附件证据应保留在凭证链路中。

### 12.3 AI 不得越权

验证标准：

- AI 不能直接提交凭证。
- AI 不能直接审核凭证。
- AI 不能直接记账凭证。
- AI 不能绕过金额平衡、科目状态、期间状态、辅助核算等校验。

### 13.2 API 边界验证

如果需要确认 403，可以使用 PowerShell 调用 API：

```powershell
$body = @{
  accountSetId = "替换为 P1-002 的账套 ID"
  fiscalYear = 2026
  periodNo = 1
  voucherDate = "2026-01-15"
  createdBy = "zhang-san"
  lines = @(
    @{ summary = "越权测试"; accountCode = "6602"; debit = 100; credit = 0 },
    @{ summary = "越权测试"; accountCode = "1002"; debit = 0; credit = 100 }
  )
} | ConvertTo-Json -Depth 5

Invoke-RestMethod `
  -Method Post `
  -Uri http://127.0.0.1:3000/vouchers `
  -Headers @{ "Actor-Id" = "zhang-san"; "Idempotency-Key" = "deny-cross-account-set-001" } `
  -ContentType "application/json" `
  -Body $body
```

预期结果：

- 返回 403。
- 错误码为 `ACCOUNT_SET_ACCESS_DENIED`。
- “平台管理 -> 操作日志”中出现 `account_set.access_denied`。

## 15. 负向规则测试清单

逐项验证下面规则：

| 场景 | 操作 | 预期 |
| --- | --- | --- |
| 凭证不平衡 | 借方 500、贷方 400 后提交 | 提交失败 |
| 分录双边金额 | 同一行同时填借方和贷方 | 保存或提交失败 |
| 分录无金额 | 同一行借贷都为空或 0 | 保存或提交失败 |
| 负数金额 | 输入 -100 | 保存或提交失败 |
| 非末级科目 | 用父级科目录凭证 | 提交失败 |
| 禁用科目 | 禁用 `6602` 后提交凭证 | 提交失败 |
| 禁止手工录入科目 | 关闭 allowManualEntry 后提交 | 提交失败 |
| 缺辅助核算 | 绑定部门但凭证未填部门 | 提交失败 |
| 制单人自审 | 张三审核自己凭证 | 审核失败 |
| 重复记账 | 对 posted 凭证再次记账 | 返回重复记账错误 |
| 已记账附件删除 | 删除 posted 凭证附件 | 删除失败 |
| 未记账期间关闭 | 有未记账凭证时关闭期间 | 关闭失败 |
| closed/locked 期间写入 | 在关闭或锁定期间新增/改凭证 | 操作失败 |
| AI 直接成正式凭证 | 只生成 AI 建议不确认 | 凭证列表不新增正式凭证 |
| 跨账套访问 | 未授权用户访问其他账套 | 403 且审计日志记录 |

## 16. 验收通过标准

完整通过 Phase 1 手工验收时，应满足：

- 完整财务闭环成功：凭证状态为 `posted`。
- 总账金额正确。
- 明细账显示该凭证。
- 科目余额表金额正确。
- 试算平衡差额为 0。
- 附件可查看，已记账附件不可删除。
- AI 只能生成草稿建议，必须人工确认后才变成正式草稿。
- 权限隔离返回 403，不能看到其他账套数据。
- 审计日志能追踪关键操作。
- 所有 Phase 1 页面可打开，无控制台错误。
