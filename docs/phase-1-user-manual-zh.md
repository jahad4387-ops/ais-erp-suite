# Phase 1 用户手册

最后更新：2026-06-09

这份手册面向第一次使用本系统的业务用户。你可以把它当成日常操作说明：先登录，再建账套和基础资料，然后录凭证、审核、记账，最后查看账簿和报表。

## 1. 系统能做什么

Phase 1 主要完成总账业务闭环：

- 管理账套和用户权限。
- 维护会计科目、辅助核算、期初余额、会计期间。
- 录入凭证，上传附件。
- 提交、审核、驳回、记账、红字冲销。
- 查询总账、明细账、科目余额表、试算平衡表。
- 导出报表。
- 做银行对账。
- 让 AI 根据业务描述生成凭证草稿，但所有正式操作仍由人工确认。
- 查看操作日志，追踪谁在什么时候做了什么。

## 2. 登录系统

打开浏览器访问：

```text
http://127.0.0.1:5173/
```

本地默认账号：

| 用户名 | 密码 | 说明 |
| --- | --- | --- |
| `system` | `system` | 管理员账号 |
| `viewer` | `viewer` | 查询账号 |

登录后会进入“财务工作台”。如果你看到账套名称、当前年度期间和待办信息，说明登录成功。

## 3. 页面布局

左侧是主菜单：

- 工作台：查看当前账套和待办。
- 平台管理：账套、用户、角色、部署配置、操作日志。
- 基础档案：科目、编码规则、辅助核算、期初余额、会计期间。
- 总账：凭证管理、录入凭证、审核工作台、记账批次、银行对账。
- 报表：试算平衡表、科目余额表、明细账等。
- Agent 中心：AI 凭证草稿。

右上方或页面上下文会显示当前账套、期间、用户等信息。录入凭证前，请先确认当前账套正确。

## 4. 账套管理

路径：平台管理 -> 账套管理

账套可以理解为一家公司或一套独立账本。不同账套之间的数据互相隔离。

新建账套时需要填写：

- 账套编码：例如 `001`。
- 账套名称：例如 `某某公司账套`。
- 公司名称。
- 本位币：通常为 `CNY`。
- 会计准则。
- 启用年度和启用期间。

保存后，系统会生成会计期间。你可以选择一个账套作为当前账套，后续科目、凭证、报表都会围绕当前账套进行。

启用账套后，部分初始化资料会被锁定，例如启用年度、启用期间、部分基础档案和期初余额。若确实要修改初始化资料，需要先停用账套。

## 5. 用户、角色和账套授权

路径：

- 平台管理 -> 角色权限
- 平台管理 -> 用户管理

系统通过“角色”控制用户能做什么，通过“账套授权”控制用户能看哪套账。

常见角色：

- 制单人：录入凭证、上传附件、提交凭证。
- 审核人：审核或驳回凭证。
- 记账人：对审核通过的凭证记账。
- 查询人：查看凭证和报表。
- 管理员：维护账套、用户、权限和基础资料。

新增用户时，需要设置用户名、姓名、初始密码和角色。新增后，再到“账套授权”页签，把用户加入指定账套。没有账套授权的用户，即使有业务权限，也不能查看或处理该账套数据。

## 6. 部署配置

路径：平台管理 -> 部署配置

这个页面主要给管理员使用，用于检查系统运行状态：

- 当前部署模式。
- 平台数据存储方式。
- 数据库是否配置。
- 附件存储是否配置。
- 外部对象存储是否就绪。
- JWT 密钥和轮换配置是否满足生产要求。

本地试用时，数据库可能显示未配置或使用内存模式，这是正常的。上线前，管理员需要按部署手册配置数据库、JWT 密钥和附件存储。

## 7. 会计科目

路径：基础档案 -> 会计科目

会计科目是凭证分录的基础。录凭证前，必须先有可用的末级科目。

你可以：

- 导入标准科目。
- 新增科目。
- 编辑科目名称、类型、余额方向、现金/银行标记。
- 禁用科目。
- 启用科目。
- 删除未使用科目。

常见字段说明：

- 科目编码：例如 `1002`。
- 科目名称：例如 `银行存款`。
- 科目类型：资产、负债、权益、收入、费用等。
- 余额方向：借方或贷方。
- 是否末级：只有末级科目能用于凭证。
- 是否允许手工录入：关闭后，该科目不能被手工凭证使用。
- 是否银行科目：银行对账会用到。

注意：

- 已经被凭证使用的科目不能随意删除。
- 禁用科目后，新凭证不能再提交使用该科目。
- 非末级科目不能直接录入凭证。

## 8. 科目编码规则

路径：基础档案 -> 科目编码规则

科目编码规则用于控制科目编码长度。常见规则是：

```text
4-2-2-2
```

这表示有效编码长度为 4、6、8、10。例如：

- `1002` 有效。
- `100201` 有效。
- `10020101` 有效。

保存规则后，新建科目时系统会按规则校验。

## 9. 辅助核算

路径：基础档案 -> 辅助核算

辅助核算用于给凭证分录增加更细的业务维度，例如部门、员工、客户、供应商、项目。

建议按三步使用：

1. 建辅助类型：例如 `department`，名称 `部门`。
2. 建辅助项目：例如 `D001 财务部`。
3. 做科目绑定：把某个科目设置为必须填写部门。

如果一个科目要求辅助核算，录凭证时必须填写对应辅助值，否则提交会失败。

## 10. 期初余额

路径：基础档案 -> 期初余额

期初余额用于录入启用账套时各科目的初始金额。

每一行通常包含：

- 科目。
- 年度。
- 期间。
- 借方期初。
- 贷方期初。

一行只能填借方或贷方，不能两边都填。页面会显示：

- 借方合计。
- 贷方合计。
- 差额。
- 是否平衡。

如果借贷不平衡，建议先查明原因，再继续正式业务。

## 11. 会计期间

路径：基础档案 -> 会计期间

会计期间控制凭证能不能录入和处理。

常见状态：

- open：开放，可以录入和处理凭证。
- closed：已关闭，不能再写入凭证。
- locked：已锁定，只读。

常见操作：

- Open：打开期间。
- Set Current：设为当前期间。
- Close：关闭期间。
- Lock：锁定期间。
- Generate P/L：生成损益结转草稿。

关闭期间前，系统会检查是否还有未记账凭证。如果还有未记账凭证，需要先审核记账，再关闭期间。

## 12. 录入凭证

路径：总账 -> 录入凭证

录入凭证时需要填写：

- 凭证日期。
- 摘要。
- 科目。
- 借方或贷方金额。
- 必要的辅助核算。
- 可选附件。

基本规则：

- 至少两行分录。
- 每行只能填借方或贷方，不能同时填。
- 金额不能为负数。
- 借方合计必须等于贷方合计。
- 科目必须是启用的末级科目。
- 如果科目要求辅助核算，必须填写辅助值。

示例：

| 摘要 | 科目 | 借方 | 贷方 |
| --- | --- | ---: | ---: |
| 购买办公用品 | 6602 管理费用-办公费 | 500 | 0 |
| 银行支付 | 1002 银行存款 | 0 | 500 |

附件区域可以选择本地文件。保存凭证时，系统会先上传附件，再把附件绑定到凭证。

## 13. 凭证管理

路径：总账 -> 凭证管理

这里可以查看和处理凭证。

常见状态：

- draft：草稿。
- submitted：已提交，等待审核。
- approved：已审核，等待记账。
- posted：已记账。
- rejected：已驳回。
- voided：已作废。

常见操作：

- 草稿或驳回凭证：可以提交。
- 已提交凭证：可以审核或驳回。
- 已审核凭证：可以记账。
- 已记账凭证：可以创建红字冲销草稿。
- 草稿、提交、驳回凭证：可以走删除即作废路径。
- 任意凭证：可展开查看分录、附件、状态日志、来源链路。

重要规则：

- 制单人不能审核自己的凭证。
- 已记账凭证不能重复记账。
- 已记账凭证附件不能删除。
- 关闭或锁定期间内的凭证不能继续写操作。

## 14. 审核工作台

路径：总账 -> 审核工作台

审核工作台适合审核人员批量处理凭证。

你可以按下面条件筛选：

- 会计期间。
- 凭证状态。
- 制单人。
- 科目或摘要关键字。

审核人员可以：

- 选择一张或多张 submitted 凭证，批量审核通过。
- 选择一张或多张 submitted 凭证，填写原因后批量驳回。

如果凭证所在期间不是 open，系统会阻止审核动作。

## 15. 记账和记账批次

单张记账：

1. 打开“总账 -> 凭证管理”。
2. 找到 approved 凭证。
3. 点击“记账”。

批量记账：

1. 在凭证管理中筛选 approved 凭证。
2. 勾选多张凭证。
3. 点击 Batch Post。

查看批次：

1. 打开“总账 -> 记账批次”。
2. 查看批次列表。
3. 打开批次详情。

批次详情会显示每张凭证成功或失败，以及失败原因。

## 16. 账簿和报表

路径：报表

报表页面包含：

- General Ledger：总账。
- Sub-ledger：明细账。
- Account Balances：科目余额表。
- Trial Balance：试算平衡表。

试算平衡表重点看三项：

- Debit：借方合计。
- Credit：贷方合计。
- Difference：差额。

如果 Difference 为 0，表示借贷平衡。

明细账中可以点击凭证号，打开凭证详情，查看凭证分录。

页面支持导出 CSV 或 Excel 兼容文件。导出内容与当前页面数据一致。

## 17. 银行对账

路径：总账 -> 银行对账

银行对账用于把银行流水和系统里的银行科目分录匹配起来。

使用步骤：

1. 确保银行科目已标记为银行科目。
2. 确保相关银行付款或收款凭证已经记账。
3. 导入银行流水。
4. 创建对账任务。
5. 点击自动匹配。

系统会按账套、期间、银行科目、日期、金额查找可匹配的记账分录。匹配成功后，流水状态会更新，并写入审计日志。

## 18. Agent 中心和 AI 草稿

路径：Agent 中心

AI 可以帮你根据业务描述生成凭证草稿建议。

示例输入：

```text
购买办公用品 500 元，银行支付
```

AI 生成后，你需要人工检查：

- 科目是否正确。
- 金额是否正确。
- 借贷方向是否正确。
- 附件和证据是否正确。
- 风险提示是否需要处理。

确认无误后，点击转换或人工确认，系统才会创建正式凭证草稿。

AI 的边界：

- AI 不能直接提交凭证。
- AI 不能直接审核凭证。
- AI 不能直接记账。
- AI 不能绕过凭证校验。

也就是说，AI 只是帮你起草，最终责任仍由人工确认。

## 19. 操作日志

路径：平台管理 -> 操作日志

操作日志用于审计追踪。你可以按用户、模块、动作筛选。

常见日志包括：

- 创建账套。
- 创建或修改科目。
- 生成期间。
- 提交、审核、驳回、记账凭证。
- 上传和绑定附件。
- AI 草稿转换。
- 银行对账。
- 越权访问被拒绝。

管理员排查问题时，建议先看操作日志。

## 20. 常见错误和处理方法

| 提示或现象 | 可能原因 | 处理方法 |
| --- | --- | --- |
| 提交凭证失败 | 借贷不平衡 | 检查借方合计和贷方合计 |
| 科目不能使用 | 科目不是末级、被禁用或禁止手工录入 | 修改科目状态或换末级科目 |
| 缺少辅助核算 | 科目绑定了辅助维度 | 补充部门、项目等辅助值 |
| 审核失败 | 制单人审核自己的凭证 | 换审核人员登录 |
| 记账失败 | 凭证未审核或已记账 | 先审核，或不要重复记账 |
| 期间写入失败 | 期间已关闭或锁定 | 打开期间，或改到 open 期间 |
| 附件删除失败 | 凭证已记账或附件在保留期 | 保留附件，不删除 |
| 看不到账套或凭证 | 没有账套授权 | 请管理员增加账套授权 |
| AI 只生成建议 | 这是设计要求 | 人工确认后才会生成正式草稿 |

## 21. 推荐日常流程

管理员初始化：

1. 创建账套。
2. 创建角色和用户。
3. 给用户做账套授权。
4. 导入或维护科目。
5. 设置辅助核算。
6. 录入期初余额。
7. 检查会计期间。

日常做账：

1. 制单人录入凭证并上传附件。
2. 制单人提交凭证。
3. 审核人审核或驳回。
4. 记账人记账。
5. 财务人员查看报表。
6. 出纳或会计做银行对账。
7. 管理员定期查看操作日志。

月末处理：

1. 确认所有凭证已记账。
2. 查看试算平衡表，确认差额为 0。
3. 生成损益结转草稿。
4. 提交、审核、记账损益结转凭证。
5. 关闭或锁定期间。

## 22. 使用建议

- 录凭证前先确认当前账套。
- 新建科目前先看编码规则。
- 凭证附件尽量在保存凭证时一起上传。
- 审核和记账尽量使用不同用户，便于审计。
- 遇到失败提示时，先看凭证状态、期间状态、科目状态和账套授权。
- 上线前请让管理员按部署手册配置数据库、JWT 密钥和附件存储。
