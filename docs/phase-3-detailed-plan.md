# Phase 3：存货、生产与成本核心闭环详细开发计划

Phase 3 的定位从纯粹的“商贸进销存”升级为 **“库存数量管理 + 生产制造转换 + 成本叠加归集 + 存货账表 + 与总账一致性”**。

最终闭环是：

```text
存货档案 (含 BOM 物料清单)
→ 仓库 / 货位 / 批次
→ 采购入库 / 销售出库 / 其他出入库 / 调拨 / 盘点
→ 生产工单 (Work Order)
→ 生产领料出库 (耗用原材料)
→ 归集直接人工与制造费用 (来自 Phase 4 薪资与折旧)
→ 月末成本分摊引擎 (Cost Allocation)
→ 产成品完工入库 (叠加总成本)
→ 成本凭证生成
→ 存货余额与总账一致
```

# 1. Phase 3 前置条件

| 前置能力   | 要求                             |
| ------ | ------------------------------ |
| 总账     | 科目、凭证、审核、记账、辅助核算已经可用        |
| 采购销售   | 采购入库、销售出库已有 source_document_id |
| 薪资固资   | (强依赖) Phase 4 提供本月厂区直接人工工资、设备折旧费。**注：Phase 3 开发验收时，这部分费用先用 Mock 接口/手工填报替代，待 Phase 4 完成后再端到端联调。** |
| 审计控制   | 已结账期间不能新增、修改、重算成本              |

Phase 3/4 二阶段成本联调门槛：

```text
阶段 A：Phase 3 自测期间
- 使用 mock_cost_inputs 或手工录入的人工/折旧成本池。
- 成本池必须带 source_type=mock/manual、account_set_id、period_no、work_order_id、cost_type、amount、locked_at。
- 月末成本计算页面必须明确显示“外部人工/折旧成本暂未接入 Phase 4 正式来源”。
- 可以验证工单分摊算法、成本凭证草稿、零数量余值结转和总账联查，但不能宣称生产成本端到端完成。

阶段 B：Phase 4 完成后
- Phase 3 必须改为消费 Phase 4 锁定后的 payroll_runs 与 depreciation_runs 输出。
- 同一期间的工单成本分配必须能追溯到工资批次、折旧批次、凭证草稿和总账余额。
- Phase 3 的最终验收必须重跑“领料 -> 工资/折旧锁定 -> 成本分配 -> 完工入库 -> 成本凭证 -> 存货/总账对账”端到端场景。
- 如果 Phase 4 输出未锁定或未对账，Phase 3 成本计算工作台只能 dry-run，不能生成正式成本凭证草稿。
```

# 4. 核心业务流程与成本防坑设计

## 4.1 生产制造成本归集流程

> [!CAUTION]
> **月末成本归集的顺序死锁**
> 加入生产制造后，月末结账必须严格遵循时间线红线：
> 1. 必须先锁死当月全部**领料单**，计算出原材料出库成本（直接材料）。
> 2. 必须等待 Phase 4 算完当月**生产工人工资**和**车间设备折旧**（归集人工与制造费用）。
> 3. 最后运行 Phase 3 的**“工单成本分配引擎”**，将这三部分成本按工时/产量比例砸进产成品的“完工入库成本”中。
> **顺序一旦颠倒，成本永远算不平。**

## 4.2 零数量金额余值结转

> [!CAUTION]
> **数量为0但金额不为0的“余值死锁”**
> 采用移动平均等方法时，只要系统判定某存货数量变为 `0`，必须自动生成一笔成本调整记录，将残余库存金额（如暂估尾差）全部结转入当期主营业务成本或损益科目，确保“数量为 0 时，金额一定为 0”。

## 4.3 批次与平均法的互斥

> [!IMPORTANT]
> **批次管理与成本计价方法的互斥**
> 凡是开启了“批次管理”的存货，成本计价方法应强制限定为“个别计价法”或“FIFO”。严禁对批次存货跑全局“移动平均法”。

# 5. 前端页面规划

Phase 3 前端遵循 `docs/ui-design.md`，页面必须让用户看清“数量、批次、成本、工单、凭证影响”之间的关系。库存和成本页面以表格、状态条、异常面板和工作台为主，不做炫技型库存可视化。

| 页面 | 设计重点 |
| --- | --- |
| 存货档案 | 展示存货分类、计量单位、批次规则、成本方法、是否自制和停用状态 |
| BOM 维护 | 左侧产品/BOM 版本树，右侧物料明细表，展示用量、损耗率、生效状态 |
| 仓库与货位 | 表格维护仓库、货位、启用状态，详情展示库存限制和负责人 |
| 库存期初 | 按存货、仓库、货位、批次录入期初数量和金额，固定显示试算结果 |
| 出入库单 | 单据头 + 明细表，展示来源订单、批次、数量、成本状态和凭证草稿 |
| 调拨单 | 调出/调入仓库并列展示，提交前检查库存不足和期间状态 |
| 盘点工作台 | 账面数、盘点数、差异数、差异原因、调整凭证预览集中展示 |
| 生产工单 | 展示计划产量、领料状态、完工状态、成本归集状态和来源 BOM |
| 领料/完工页面 | 明细表展示领用材料、完工产品、批次和成本状态 |
| 成本计算工作台 | 展示月末成本顺序检查、未锁单据、批次互斥错误、零数量余值和凭证影响 |
| 收发存查询 | 期间、存货、仓库、批次筛选；支持从余额下钻到出入库单和凭证 |

UI 验收：

- 成本计算前必须展示前置检查，不允许用户在红灯状态下直接执行。
- 批次存货和移动平均互斥必须在存货档案和成本工作台同时提示。
- 零数量金额余值必须在成本计算结果里单独列出，并说明结转凭证影响。
- 存货明细、库存余额、成本凭证必须可以互相联查。

# 6. 开发排期：12 周详细版

- 第 1-3 周：存货档案、批次序列号、BOM结构设计、库存期初。
- 第 4-6 周：常规出入库、调拨、盘点、移动平均/FIFO核心算法。
- 第 7-9 周：生产工单、领料出库、完工入库逻辑打通。
- 第 10-11 周：打通 Phase 4 费用（前期使用 Mock 数据），实现月末生产成本分摊引擎与“零数量余值结转”。
- 第 12 周：生成总账成本凭证、库存明细账对账、端到端联调验收。

# 7. 一句话总结

升级后的 Phase 3 完美折叠了“商贸进销存”与“工业制造成本”：

```text
准确的库存数量
+ 严密的工单领料完工转换
+ 融合薪资与折旧的制造成本分摊
+ 防坑到底的出库算法兜底
= 毫厘不差的业财一体化存货总账
```
