# 实施路线图

## Phase 0: 需求冻结与验证资产

- 完成功能追踪矩阵。
- 建立 PRD、架构、Agent 操作规范。
- 建立最小领域层测试，确保模块目录和凭证平衡规则可验证。

交付物：

- `docs/prd.md`
- `docs/traceability-matrix.md`
- `docs/architecture.md`
- `docs/agent-operations.md`
- `packages/domain`

## Phase 1: 平台与总账 MVP

- 账套、组织、期间、用户、权限。
- 科目、辅助核算、凭证、审核、记账。
- 总账、明细账、科目余额表、试算平衡。
- AI 记账草稿接口。

验收：

- 从期初到凭证、记账、账簿、报表取数闭环。
- AI 只能生成草稿，不能直接记账。

## Phase 2: 采购付款 + 销售收款

- P2P：采购订单、入库、发票、应付、付款、核销。
- O2C：销售订单、发货、发票、应收、收款、核销。
- 应收应付账龄、付款计划、回款分析。

验收：

- 采购和销售业务自动生成总账凭证。
- 往来余额与总账辅助账一致。

## Phase 3: 存货与成本

- 存货、仓库、计量、批次。
- 出入库、盘点、调拨。
- 移动平均、FIFO、个别计价、月末一次加权。
- 收发存、库龄、周转分析。

验收：

- 出库成本计算可复核。
- 存货明细账、库存余额和总账一致。

## Phase 4: 薪资与固定资产

- 工资项目、工资计算、工资分摊凭证。
- 固定资产卡片、折旧、转移、处置。

验收：

- 工资费用、折旧费用自动生成凭证。
- 资产净值、累计折旧、总账余额一致。

## Phase 5: 报表与管理分析

- 报表模板和公式引擎。
- 资产负债表、利润表、现金流量表。
- 采购、销售、存货、应收应付管理报表。
- 报表导出和 AI 解读。

验收：

- 报表取数可追溯到科目余额和明细账。
- 报表锁定后不可被无权限修改。

## Phase 6: Agent 工作流与生产化

- Agent API、dry-run、审批、审计。
- 结账助手、对账助手、催收助手。
- 迁移工具、备份恢复、安全测试、性能测试。

验收：

- Agent 高风险动作必须审批。
- 所有正式数据变更都有完整审计链。
