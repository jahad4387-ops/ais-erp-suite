# 实施路线图

## Phase 0: 需求冻结与验证资产

- 完成功能追踪矩阵。
- 建立 PRD、架构、Agent 操作规范。
- 建立内网与公网兼容部署设计。
- 建立 UI 与页面设计规划，明确商务简洁风格、全局导航、页面模式和各 Phase 页面入口。
- 建立最小领域层测试，确保模块目录和凭证平衡规则可验证。
- 审核参考书，补充安装/卸载、数据权限、单据设置、模块初始化、银行对账、UFO 报表和实验验收遗漏项。

## Phase 1: 平台与总账 MVP

- 账套、组织、期间、用户、权限。
- 部署模式配置、中心数据库连接、附件存储配置。
- 安装/卸载检查、系统参数、备份恢复、基础档案初始化顺序。
- 功能权限、数据范围权限、字段权限、Agent 工具权限。
- 单据编号规则、单据模板、状态流、打印/导出模板。
- 科目、辅助核算、凭证、审核、记账。
- 总账、明细账、科目余额表、试算平衡、银行对账。
- AI 记账草稿接口。
- 按 `docs/ui-design.md` 落地 Web 外壳、模块导航、账套/期间上下文、凭证和账簿核心页面。

验收：

- 从期初到凭证、记账、账簿、报表取数闭环。
- 可完成总账初始化、日常业务、银行对账、期末处理四个实验式场景。
- AI 只能生成草稿，不能直接记账。
- 两个用户通过同一中心服务访问时，凭证和附件状态能同步。

## Phase 2: 采购付款 + 销售收款

- P2P：采购订单、入库、发票、应付、付款、核销。
- O2C：销售订单、发货、发票、应收、收款、核销。
- 应收应付账龄、付款计划、回款分析。

验收：

- 采购和销售业务自动生成总账凭证。
- 往来余额与总账辅助账一致。
- 采购、销售、应收、应付和核销页面具备列表、详情、状态流、凭证预览和联查入口。

## Phase 3: 存货、生产与成本

- 存货档案、BOM 物料清单、仓库、计量、批次。
- 生产工单、领料出库、产成品完工入库、盘点、调拨。
- 移动平均、FIFO、个别计价、月末一次加权。
- 先用 mock/manual 成本池验证人工和折旧分摊接口，Phase 4 完成后必须二次联调替换为正式工资/折旧成本来源。
- 收发存、库龄、周转分析。

验收：

- 出库成本计算可复核。
- 存货明细账、库存余额和总账一致。
- 存货、BOM、工单、盘点和成本计算工作台可以展示数量、成本、异常和凭证影响。
- Phase 3 最终验收必须跑通“领料 -> Phase 4 工资/折旧锁定 -> 成本分配 -> 完工入库 -> 成本凭证 -> 存货/总账对账”。

## Phase 4: 薪资与固定资产

- 工资项目、工资计算、工资分摊凭证。
- 固定资产卡片、折旧、转移、处置。
- 向 Phase 3 输出已锁定、可追溯、可对账的生产人工成本池和车间设备折旧成本池。
- 工资期末处理、固定资产期末处理和与总账对账。

验收：

- 工资费用、折旧费用自动生成凭证。
- 资产净值、累计折旧、总账余额一致。
- 工资批次、工资调整、资产卡片、折旧计提和对账页面具备人工复核与审计痕迹。

## Phase 5: 报表与管理分析

- 报表模板和公式引擎。
- 资产负债表、利润表、现金流量表。
- UFO 报表式自由格式模板、单元格公式、模板复制、报表输出。
- 采购、销售、存货、应收应付管理报表。
- 报表导出和 AI 解读。

验收：

- 报表取数可追溯到科目余额和明细账。
- 报表锁定后不可被无权限修改。
- UFO 报表设计器、报表运行、管理分析和 AI 解读页面符合简洁商务风格，不做炫技型大屏。

## Phase 6: Agent 工作流与生产化

- Agent API、dry-run、审批、审计。
- 结账助手、对账助手、催收助手。
- 迁移工具、备份恢复、安全测试、性能测试。
- 内网部署、私有云/VPN 部署、公网 HTTPS 部署的配置和验收脚本。

验收：

- Agent 高风险动作必须审批。
- 所有正式数据变更都有完整审计链。
- 同一套代码可在局域网和公网部署拓扑下运行。
- Agent 动作、审批、证据链和 Replay 页面清楚区分建议、dry-run、审批和正式执行。
