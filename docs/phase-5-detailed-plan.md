# Phase 5：报表与管理分析详细开发计划

这一阶段承接 Phase 1～4 的所有核算数据，目标是把总账、购销、存货、薪资、固定资产的数据统一沉淀到 **报表模板、公式引擎、报表运行、报表审核锁定、导出、AI 解读** 这一整套报表体系里。

# Phase 5 总目标

Phase 5 要建立一个既能做法定财务报表，又能做管理分析报表的综合报表中心：

```text
总账余额 / 明细账
+ 采购、销售、存货、薪资、固定资产子系统数据
→ 报表公式引擎
→ 报表模板
→ 报表运行结果
→ 报表审核
→ 报表锁定
→ Excel / PDF 导出
→ AI 解读与经营分析
```

最终要完成三类核心报表：
1. **法定财务报表**：资产负债表、利润表、现金流量表、所有者权益变动表。
2. **UFO 式自由格式报表**：自由单元格、公式、格式、模板复制、报表输出。
3. **管理分析报表**：采购、销售、存货、账龄、现金流、人工、折旧、整体经营分析。

# 1. Phase 5 前置条件与模块契约

推荐使用 `feature/reporting` 分支推进。

```text
模块名称：reporting

负责范围：
报表模板、自由格式报表设计器、公式引擎、报表运行快照、报表审核、报表锁定、报表导出、法定财务报表、管理分析报表、报表取数单据级追溯、AI 报表解读。

不负责范围：
原始业务单据录入、凭证生成、期末结账动作。

会调用的核心服务：
科目余额表、明细账引擎、辅助核算聚合接口、权限校验、结账期间校验、审计服务。
```

# 2. 核心业务流程与防坑设计

## 2.1 报表模板设计流程
创建报表模板 → 设置行列/单元格/合并 → 配置公式 → 校验 → 保存版本 → 发布。已发布的版本如果修改，必须生成新版本。

## 2.2 报表编制流程与“幽灵数据”隔离

> [!CAUTION]
> **未记账凭证的隔离红线**
> 报表运行时，公式引擎必须提供 `include_unposted` 参数。**法定财务报表在正式生成时，系统强制过滤未记账的草稿凭证**；管理报表或内部测算时可按需开启“模拟模式”。决不允许未记账的幽灵数据污染对外公布的正式财报。

## 2.3 报表重算流程与“绝对死锁”

> [!IMPORTANT]
> **期间结账与报表重算的物理隔离**
> 这是合规的绝对底线：无论一张报表当前是否处于 `locked` 状态，只要其所属的会计期间在总账里已经被标记为 `Closed`（已结账），系统**绝对禁止该报表进行任何公式引擎重算（Recalculate）**。已结账期间的法定报表如同刻在石头上。

## 2.4 锁定快照的“断连”渲染

> [!NOTE]
> **报表锁定的“快照”物理隔离**
> 当报表状态流转到 `locked`（已锁定）或 `archived` 时，系统前端展现与文件导出服务，**必须且只能依赖数据库表里的 `calculated_value` 和 `display_format` 渲染**。此时系统强制“熔断”对底层公式引擎和科目余额表的调用。这样确保十年后的查账，哪怕当初的科目早已被删，旧报表也依然在一秒内一字不漏地原样呈现。

## 2.5 单元格追溯流程 (Data Lineage)
点击单元格 → 查看公式 → 查底层科目/辅助余额 → 查明细账 → 查对应凭证 → 查来源单据/附件。报表系统的灵魂在于“可解释、可查证”。

# 3. 报表公式引擎性能防坑

支持的公式：`BAL()`, `AMT()`, `OPENING()`, `CLOSING()`, `AUX_BAL()`, 跨期与跨表页单元格引用。

> [!WARNING]
> **严防 N+1 查询导致的性能雪崩**
> 公式引擎严禁采用“解释一条公式查一次库”的低效做法！后端解析器必须先提取整张表页所有公式的 `dependencies_json`，执行**批量预查询 (Batch Fetching)**，将本期涉及的所有科目/辅助核算余额一次性拖入内存 HashMap，然后纯在内存中完成 AST 树运算，保障百行千列大表也能在毫秒级算完。

# 4. 法定报表设计要求

## 4.1 资产负债表与利润表
通过科目与辅助核算余额的合并抵消完成，支持差额定位预警。

## 4.2 现金流量表的“不平衡兜底”

> [!CAUTION]
> **未分配现金流强制曝光**
> 业务在制单时极易忘记分配现金流辅助项。报表引擎在计算时，必须主动抓取“涉及现金银行科目，但未被挂载任何现金流项目标签”的凭证金额。如果该金额不为 0，系统必须**直接拦截并高亮显示为“未分配异常现金流”**，强迫制单人补录，否则直接拒绝报表提交与锁定。

## 4.3 UFO 式自由格式报表 (Excel-like)
支持类似传统 UFO 的多表页、合并单元格、边框、字体、背景色。并且这套 UI 渲染引擎与法定报表共享。

# 5. 管理分析与 AI 报表解读

## 5.1 管理分析大屏
覆盖采购趋势、销售毛利与回款、存货周转与呆滞、应收应付账龄、部门人工成本、闲置资产清单等。

## 5.2 AI 解读输出架构
Agent 原生设计中，解释报表属于低风险。
输出 JSON 结构需包含：经营摘要、核心指标异动发现 (`keyFindings`)、业务风险提示 (`warnings`)、以及不可或缺的证据链引用 `evidenceRefs` (必须精确指向 `report_cell:BS!B12` 或具体指标)。

---

# 6. 数据模型设计核心

## 6.1 模板与单元格配置
- `report_templates`, `report_template_versions`
- `report_sheets`, `report_cells` (存储布局、格式、公式、是否可编辑)
- `report_formulas` (存储公式 AST 与预提取依赖)

## 6.2 报表运行快照与追溯
- `report_runs` (记录运行时参数、账套、期间、运行状态、数据快照 Hash)
- `report_run_cells` (核心：存储 `calculated_value` 与 `trace_id`)
- `report_trace_links` (存储由 `trace_id` 关联的底层总账或业务单据行来源，供无缝下钻)

## 6.3 导出与分析
- `report_exports` (文件导出历史，含安全审计日志)
- `ai_report_interpretations` (AI 分析草稿结果)

# 7. 前端页面规划

Phase 5 前端遵循 `docs/ui-design.md`。报表与管理分析页面要保持商务简洁，重点是可追溯、可审核、可锁定和可导出。管理分析不做纯展示型大屏，优先提供指标、趋势、异常和明细下钻。

| 页面 | 设计重点 |
| --- | --- |
| 报表模板列表 | 展示模板类型、版本、发布状态、适用期间、权限和最近运行 |
| UFO 报表设计器 | 类 Excel 网格、公式栏、表页标签、格式工具栏、单元格属性和校验面板 |
| 公式校验面板 | 展示公式 AST、依赖科目、辅助核算、跨期引用和错误定位 |
| 报表运行 | 参数选择、运行状态、异常单元格、未记账隔离提示和重算限制 |
| 报表详情 | 锁定快照只读展示，单元格可查看公式、来源余额、明细账和凭证 |
| 报表审批 | 待审批队列、差额异常、现金流未分配提示、审批意见和锁定动作 |
| 管理分析 | 采购、销售、存货、账龄、现金流、人工、折旧等指标和明细下钻 |
| AI 报表解读 | 经营摘要、关键发现、风险提示和证据引用，人工确认后归档 |
| 导出中心 | Excel/PDF 导出历史、下载权限、敏感字段提示和审计日志 |

UI 验收：

- 已锁定或归档报表只能依赖快照渲染，页面不得触发公式重算。
- 点击单元格必须能查看公式、取数来源、明细账、凭证和来源单据。
- 现金流量表未分配现金流必须高亮并阻止提交与锁定。
- AI 解读必须展示证据引用，不能只展示自然语言结论。
- 管理分析页面必须能从指标下钻到业务单据或账簿明细。

# 8. 开发排期：12 周详细版

- 第 1-3 周：数据底座（模块库搭建）、UFO 式自由格式报表设计器第一版（前端类似简易 Excel 表格引擎）。
- 第 4-5 周：公式解析器与核心引擎搭建（实现 AST 拆解与 Batch Fetching 性能优化）。
- 第 6-7 周：资产负债表与利润表模板、总账取数接入、单元格底层追溯机制打通（`trace_links`）。
- 第 8 周：现金流量表研发（加入“未分配现金流拦截”强制校验）、所有者权益变动表。
- 第 9-10 周：报表审批流转、期间死锁控制（Closed期间禁算）、静态快照隔离渲染改造、Excel/PDF 导出服务。
- 第 11 周：八大管理报表大屏对接（采购销售存货等）、AI 经营分析与预警对接。
- 第 12 周：权限拦截测试、导出的敏感日志审计测试、端到端联调（Phase 1 至 Phase 5 全链路跑通验证）。

# 9. 一句话总结

Phase 5 的核心是把前面所有核算模块沉淀成 **可追溯、可审核、可锁定、可导出、可解释** 的报表体系：

```text
总账和子系统数据 
→ 批量抓取高性能引擎 
→ 法定报表(严锁防坑) + 自由报表 + 管理分析 
→ 物理快照锁定 
→ 导出归档 
→ AI 解析决策
```

至此，我们的财务系统从纯粹的“自动化记账工具”，升维成了企业不可或缺的**“智能经营决策大脑”**。
